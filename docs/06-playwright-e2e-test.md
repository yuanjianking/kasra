# Playwright 前端页面集成测试用例

**分类:** 前端全量集成测试  
**设计原则:** 模拟真实用户操作流程，从登录到各页面功能验证

---

## A. 登录页

### TC-PW-01: 输入有效的 API Key 登录成功

- **测试观点:** 用户输入正确的 API Key 后成功进入 Dashboard
- **测试步骤:**
  1. 打开登录页
  2. 输入有效 API Key
  3. 点击登录按钮
  4. 验证 URL 变为 /dashboard
- **预期:** 登录成功

### TC-PW-02: 输入无效 API Key 显示错误

- **测试观点:** 错误的 API Key 停留在登录页并显示错误提示
- **测试数据:** 无效 Key
- **预期:** 错误信息可见，未跳转

### TC-PW-03: 未登录访问受保护页面

- **测试观点:** 清除 localStorage 后访问 /dashboard 被重定向到登录页
- **预期:** 显示登录页

## G. 端到端场景

### TC-PW-27: 完整用户操作路径

- **测试观点:** 用户从登录开始，完成查看 Dashboard → 管理规则 → 查看审计日志 → 查看用户行为的完整流程
- **测试步骤:**
  1. 登录
  2. Dashboard 查看概览
  3. 规则管理 - Input tab - 禁用一条规则 - 验证状态变化
  4. 切换到 Code Review tab - 查看规则列表
  5. 审计日志 - 查看列表 - 筛选 P0
  6. 用户行为 - 查看列表
- **预期:** 所有页面加载正常，所有交互可用
